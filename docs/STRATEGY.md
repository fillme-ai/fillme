# Product Strategy: OpenWhip

**Date**: 2026-04-15
**Stage**: Idea → MVP 준비 단계

---

## 1. Vision

> **"이력서는 한 번만 쓰세요. 나머지는 우리가 채워드립니다."**

구직이라는 인생의 중요한 순간에서, 사람들이 반복 입력이 아닌 **자기소개와 준비**에 집중할 수 있는 세상을 만든다. 궁극적으로 구직자의 모든 채용 여정을 하나의 대시보드에서 관리하는 **구직자의 컨트롤 타워**가 된다.

---

## 2. 핵심 문제 정의

사람인/잡코리아 같은 채용 플랫폼에는 이력서를 한 번 등록하면 그걸로 지원이 가능하다. **진짜 문제는 기업 자체 채용 사이트**에 있다.

- 삼성 커리어스, 현대 채용, SK 채용, 네이버 채용, 쿠팡 채용 등 **기업마다 자체 채용 사이트**가 있음
- 이 사이트들은 양식이 전부 다르고, "이력서 가져오기" 기능이 **없음**
- 대기업/공기업 지원 시 입사지원서가 매우 길고 복잡 (병역, 어학, 자격증, 수상 이력, 자기소개서 등)
- 공채 시즌에 10~20개 기업에 동시 지원하면 **며칠을 입력에만 소모**

**솔루션**: 이력서 업로드 → AI 파싱 → 크롬 확장으로 **어떤 채용 사이트든** 폼을 자동 인식하고 입력

---

## 3. Target Segments

| Segment | 규모 | Pain Level | 현재 대안 | Priority |
|---------|------|-----------|----------|----------|
| 공채 준비 신입 구직자 (20대) | ~50만명/년 | **극도로 높음** — 10~20개 기업 동시 지원, 기업마다 긴 양식 | 수동 입력, 복사-붙여넣기 | **P0** |
| 적극적 이직 준비자 (30대) | ~80만명/년 | **높음** — 기업 채용 사이트 직접 지원 빈번 | 수동 입력 | **P0** |
| 프리랜서/계약직 | ~20만명 | 중간 | 직접 연락, 포트폴리오 | P2 |
| 대학교 취업지원센터 (B2B) | ~200개교 | 중간 | 엑셀 관리 | P2 |

**Primary segment**: **공채 준비 신입 구직자** — 10~20개 기업에 동시 지원하는 시즌에 가장 극심한 고통. 바이럴 가능성도 가장 높음.

**Explicitly not serving**:
- 사람인/잡코리아 내에서만 지원하는 사용자 (플랫폼 자체 기능으로 충분)
- 해외 취업 준비자 (한국 기업 채용 사이트 특화 우선)

## 4. Value Propositions (JTBD)

**공채 준비생**:
> "공채 시즌에 10개 이상 기업에 지원할 때, 매번 같은 정보를 다시 입력하지 않고 싶다. 그래야 자기소개서 작성과 면접 준비에 시간을 쓸 수 있으니까."

**이직 준비자**:
> "관심있는 기업의 자체 채용 사이트에서 지원할 때, 이력서 정보를 원클릭으로 채우고 싶다. 그래야 퇴근 후 짧은 시간에도 지원을 완료할 수 있으니까."

## 5. Strategic Trade-offs

| We Choose | Over | Because |
|-----------|------|---------|
| **기업 자체 채용 사이트 타겟** | 사람인/잡코리아 등 채용 플랫폼 | 플랫폼은 자체 이력서 기능이 있어 증분 가치가 낮음. 기업 채용 사이트가 진짜 페인 포인트 |
| **범용 AI 폼 매퍼** | 사이트별 하드코딩 | 기업 채용 사이트는 수백~수천 개. 하드코딩 불가. AI가 폼 label을 읽고 자동 매핑하는 범용 엔진이 필수 |
| **크롬 확장 (데스크톱)** | 모바일 앱 | 채용 사이트 폼에 직접 접근해야 하므로 브라우저 확장이 유일한 방법 |
| **파싱 후 사용자 검토 필수** | 완전 자동 제출 | 이력서는 고위험 문서. 잘못된 입력은 합격/불합격을 좌우 |
| **B2C 먼저** | B2B 먼저 | 유저 기반 없이 B2B 협상력 없음 |
| **한국 시장 집중** | 글로벌 확장 | 한국 기업 채용 사이트 + 한국어 이력서 특화가 핵심 |

## 6. Key Metrics

**North Star Metric**: **주간 자동 입력 성공 횟수 (Weekly Successful Auto-fills)**

**Input Metrics**:
| 레버 | 지표 | 초기 목표 |
|------|------|----------|
| 획득 | 크롬 익스텐션 주간 신규 설치 수 | 500/주 |
| 활성화 | 설치 → 이력서 업로드 전환율 | 60% |
| 핵심 가치 | 업로드 → 첫 자동 입력 성공 전환율 | 70% |
| 리텐션 | 7일 내 2회 이상 자동 입력 사용률 | 40% |
| 수익 | Free → Paid 전환율 | 5% |

**Health Metrics**:
- 파싱 정확도 >= 90%
- 자동 입력 실패율 < 10%
- 개인정보 관련 민원 건수 = 0

## 7. Growth Engine

**Primary: 커뮤니티 기반 오가닉 성장**

1. **공채 시즌 바이럴**: "10개 기업 입사지원 30분 만에 끝냄" → 블라인드/에브리타임/오픈카톡에서 자발적 공유
2. **숏폼 콘텐츠**: 기업 채용 사이트 자동 입력 타임랩스 → 유튜브 Shorts/틱톡/릴스
3. **레퍼럴**: "친구 초대 시 무료 자동입력 3회 추가" → 구직은 친구끼리 함께 준비하는 활동
4. **대학교 채널**: 취업특강/박람회에서 무료 배포
5. **Chrome 웹스토어 SEO**: "입사지원 자동입력", "채용 폼 자동완성" 키워드 선점

## 9. Defensibility

| 방어 수단 | 유형 | 강도 | 설명 |
|-----------|------|------|------|
| **범용 AI 폼 매퍼 기술** | 기술 축적 | 중 → 강 | 수백 개 기업 채용 사이트에서 학습한 폼 매핑 노하우. 후발주자가 따라오려면 동일한 학습 과정 필요 |
| **한국어 이력서 파싱 데이터셋** | 데이터 | 중 → 강 | 사용자 증가에 비례하여 정확도 자동 향상 |
| **사용자 전환 비용** | Switching cost | 중 | 마스터 이력서 + 지원 이력 축적 시 이탈 비용 증가 |
| **플랫폼 복제 리스크 없음** | 구조적 이점 | 강 | 기업 자체 채용 사이트는 사람인/잡코리아가 통제 불가. 플랫폼이 이 기능을 복제할 수 없음 |

## Strategic Risks

| # | 리스크 | Kill 확률 | 대응 |
|---|--------|----------|------|
| 1 | 범용 AI 폼 매퍼의 정확도가 충분하지 않음 (기업마다 양식이 너무 다양) | 25% | Phase 0 기술 PoC에서 주요 기업 10곳 테스트 |
| 2 | 기업 채용 사이트가 자동 입력을 차단 (reCAPTCHA 등) | 20% | "사용자 보조 도구" 포지셔닝. 기업 채용 사이트는 상대적으로 관대 |
| 3 | 구직자 지불 의향 낮음 + 단기 사용 (낮은 LTV) | 20% | 건당 과금 + 시즌패스 모델. B2B 수익으로 보완 |

## 경쟁 환경

| 서비스 | 차이점 |
|--------|--------|
| 사람인/잡코리아 자체 기능 | 자사 플랫폼 내에서만 작동. 기업 자체 채용 사이트는 커버 불가 |
| Chrome 자동완성 | 단순 텍스트 필드만 가능. 드롭다운/날짜/복합 양식 불가 |
| Simplify (미국) | 한국 기업 채용 사이트 미지원. 한국어 이력서 파싱 불가 |
| ChatGPT/Claude 활용 | 이력서 작성은 도와주지만 폼 자동 입력은 불가 |

**핵심 차별점**: 한국 기업 자체 채용 사이트에 범용으로 대응하는 AI 자동 입력 도구는 현재 **존재하지 않음**

|------|------|-----------|

| Basic | 월 4,900원 | 무제한 파싱, 자동입력 월 30회 |
