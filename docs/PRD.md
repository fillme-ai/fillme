# PRD: OpenWhip - AI 이력서 자동 입력 서비스

**Version**: v1.0
**Date**: 2026-04-15
**Status**: Draft

---

## 1. Problem Statement

구직자는 사람인/잡코리아 같은 채용 플랫폼에는 이력서를 한 번 등록하면 그걸로 지원할 수 있다. 하지만 **기업 자체 채용 사이트**(삼성 커리어스, 현대 채용, SK 채용, 네이버 채용, 쿠팡 채용 등)에 지원할 때는 매번 처음부터 다시 입력해야 한다.

- 기업 자체 채용 사이트는 **수백~수천 개**이며, 양식이 전부 다름
- "이력서 가져오기" 기능이 **없음**
- 대기업/공기업 입사지원서는 매우 길고 복잡 (인적사항, 학력, 경력, 병역, 어학, 자격증, 수상 이력, 자기소개서 등)
- 공채 시즌에 10~20개 기업에 동시 지원하면 **며칠을 입력에만 소모**

## 5. Core Features (MVP Scope)

### P0 - Must Have (MVP 필수)

| # | 기능 | 설명 |
|---|------|------|
| F1 | **PDF 이력서 파싱** | PDF 업로드 → AI가 구조화 데이터로 변환 (이름, 연락처, 학력, 경력, 자격증, 어학, 병역 등) |
| F2 | **파싱 결과 확인/수정 UI** | 파싱된 데이터를 사용자가 확인하고 오류를 수정할 수 있는 화면. AI 확신도 낮은 필드 하이라이트 |
| F3 | **범용 AI 폼 매퍼** | 크롬 확장이 **어떤 채용 사이트든** 폼 필드의 label/placeholder를 AI로 분석하여 이력서 데이터와 자동 매핑 |
| F4 | **크롬 익스텐션** | Manifest V3 기반. 채용 사이트 감지 → "자동 입력" 버튼 표시 → 매핑 미리보기 → 입력 실행 |

### P1 - Should Have (MVP 직후)

| # | 기능 | 설명 |
|---|------|------|
| F5 | **이미지 이력서 파싱** | OCR + AI로 스캔본/캡처 이력서 지원 |
| F6 | **입력 전 미리보기** | 자동 입력 전 "이 항목이 여기에 들어갑니다" 시각적 미리보기 |
| F7 | **드롭다운/라디오 버튼 매핑** | 텍스트 필드뿐 아니라 선택형 필드(학력 구분, 병역 구분 등)도 자동 선택 |

### P2 - Nice to Have (이후 확장)

| # | 기능 | 설명 |
|---|------|------|
| F8 | **HWP 파일 파싱** | 한글 파일 포맷 지원 |
| F9 | **이력서 버전 관리** | 직군/회사별 다른 버전 저장 및 선택 |
| F10 | **지원 이력 대시보드** | 어디에 언제 지원했는지 자동 추적 |
| F11 | **자기소개서 자동 입력** | 자기소개서 항목까지 확장 |

## 6. User Stories

### US-1: 이력서 업로드 및 파싱
> **As a** 구직자,
> **I want to** 내 PDF 이력서를 업로드하면 AI가 자동으로 정보를 추출해주길,
> **So that** 기업 채용 사이트에 수동 입력할 필요가 없다.

**Acceptance Criteria**:
- [ ] 10MB 이하 PDF 파일 업로드 가능
- [ ] 파싱 완료까지 30초 이내
- [ ] 최소 추출 필드: 이름, 이메일, 전화번호, 주소, 학력(학교명, 전공, 기간, 학점), 경력(회사명, 직무, 기간, 주요 업무), 자격증, 어학 점수, 병역사항
- [ ] 한글/영문 이력서 모두 지원

### US-2: 파싱 결과 확인 및 수정
> **As a** 구직자,
> **I want to** AI가 추출한 정보를 확인하고 잘못된 부분을 수정하길,
> **So that** 정확한 정보가 기업 채용 사이트에 입력된다.

**Acceptance Criteria**:
- [ ] 모든 추출 필드가 편집 가능한 폼으로 표시
- [ ] AI 확신도가 낮은 필드(80% 미만)는 시각적 하이라이트
- [ ] 학력/경력/자격증 항목 추가/삭제 가능
- [ ] 수정 후 저장 즉시 반영

### US-3: 기업 채용 사이트에서 자동 입력
> **As a** 구직자,
> **I want to** 기업 채용 사이트에서 버튼 하나로 내 정보가 자동 입력되길,
> **So that** 40분 이상 걸리던 입력을 3분 안에 끝낼 수 있다.

**Acceptance Criteria**:
- [ ] 채용 사이트의 입력 폼을 AI가 자동 감지
- [ ] 폼 필드의 label/placeholder를 분석하여 이력서 데이터와 자동 매핑
- [ ] "자동 입력" 버튼 클릭 시 매핑된 필드에 데이터 입력
- [ ] 드롭다운/라디오 버튼 등 비텍스트 필드도 올바르게 선택
- [ ] 자동 입력 후 사용자가 검토할 수 있도록 제출은 하지 않음
- [ ] 매핑 불가능한 필드는 건너뛰고 사용자에게 목록으로 알림

### US-4: 익스텐션 인증
> **As a** 구직자,
> **I want to** 크롬 익스텐션에서 로그인하면 이력서 데이터가 바로 사용 가능하길,
> **So that** 추가 설정 없이 바로 자동 입력을 사용할 수 있다.

**Acceptance Criteria**:
- [ ] 구글/카카오 소셜 로그인 지원
- [ ] 로그인 후 저장된 이력서 데이터 자동 로드
- [ ] 로그아웃 시 로컬 데이터 완전 삭제

## 7. Technical Requirements

### 시스템 아키텍처

```
[사용자 브라우저]
  ├── 웹 대시보드 (이력서 업로드/확인/수정)
  └── 크롬 익스텐션 (폼 감지 + AI 매핑 + 자동 입력)
        ↕
[백엔드 API 서버]
  ├── 인증 서비스
  ├── 이력서 파싱 서비스 (LLM 기반)
  ├── 범용 폼 매퍼 AI (폼 label 분석 → 필드 매핑)
  └── 사용자 데이터 저장소 (암호화)
```

### 핵심 기술 요건

| 영역 | 요건 |
|------|------|
| **이력서 파싱** | PDF 텍스트 추출 + LLM 기반 구조화. 한국 특유 필드(병역, 어학, 학점) 포함 |
| **범용 폼 매퍼** | 사이트별 하드코딩 없이, AI가 폼 필드의 label/context를 읽고 이력서 필드와 매핑. 이것이 기술적 핵심 |
| **크롬 익스텐션** | Manifest V3 기반. Content Script로 DOM 접근. JS 렌더링 페이지도 접근 가능 |
| **보안** | 개인정보 전송 TLS, 저장 AES-256 암호화. 개인정보처리방침 필수 |
| **인프라** | 클라우드(AWS/GCP). 파싱 비동기 처리 |

## 10. Risks & Mitigations

| 리스크 | 영향 | 대응 |
|--------|------|------|
| **범용 폼 매퍼 정확도 부족** | 핵심 가치 훼손 | Phase 1 PoC에서 10개 기업 테스트. 실패 시 하이브리드 접근(AI + 주요 기업 어댑터) |
| **기업 채용 사이트 reCAPTCHA/봇 차단** | 기능 무력화 | "사용자 보조 도구" 포지셔닝. 입력은 사용자 트리거, 제출은 사용자가 직접 |
| **개인정보 유출** | 신뢰 상실, 법적 제재 | 최소 수집, 암호화, 로컬 처리 옵션 검토. 개인정보보호법 준수 |
| **LLM API 비용 증가** | 수익성 악화 | 파싱 결과 캐싱, 경량 모델 단계적 적용 |
| **구직자 유료 전환율 낮음** | 수익 모델 실패 | 건당 과금 병행, B2B 수익으로 보완 |
